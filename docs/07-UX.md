# 07 - Documento de Experiencia de Usuario (UX) - BeautySpot SaaS

## Resumen

Este documento define la experiencia de usuario para BeautySpot, plataforma SaaS
multi-tenant para barberías, salones de belleza y spas. Cubre los flujos principales,
la estrategia móvil, la navegación por rol, el sistema de diseño y las consideraciones
de accesibilidad.

---

## Principios de Diseño

1. **Mobile-first**: Toda interfaz se diseña primero para dispositivos tactiles y luego
   se adapta a escritorio
2. **Simplicidad**: Máximo 3 clics para completar cualquier acción principal
3. **Contexto visual**: Usar colores, iconos y jerarquía para guiar al usuario sin
   depender solo del texto
4. **Feedback inmediato**: Toda acción del usuario recibe respuesta visual instantánea
5. **Consistencia**: Patrones de interacción uniformes en todo el producto
6. **Confianza**: Transparencia en precios, tiempos y disponibilidad

## Flujo de Reserva (Perspectiva del Cliente)

### Journey Completo

```
Buscar negocio -> Ver perfil -> Seleccionar servicio -> Seleccionar profesional
-> Elegir fecha/hora -> Confirmar datos -> Recibir confirmación
```

### Paso 1: Búsqueda de Negocio

**Pantalla**: Home / Marketplace

- Barra de búsqueda prominente con placeholder: "Busca barberías, salones o spas"
- Filtros accesibles desde la pantalla inicial:
  - Ubicación (usar GPS o escribir dirección)
  - Radio de distancia: 1km, 5km, 10km, 25km
  - Tipo de negocio: Barbería, Salón de Belleza, Spa
  - Servicios específicos
- Resultados en tarjetas con:
  - Foto principal del negocio
  - Nombre y tipo
  - Calificación promedio (estrellas)
  - Distancia al usuario
  - Precio desde (servicio más económico)
  - Indicador de "Disponible hoy"
- Vista alternativa de mapa interactivo con pines
- Ordenamiento por: Relevancia, Distancia, Calificación, Precio

**Interacción móvil**: Tactil con swipe horizontal para ver fotos, tap para expandir

### Paso 2: Perfil del Negocio

**Pantalla**: `/negocio/{slug}`

- Header con foto de portada (80vh en móvil) y nombre superpuesto
- Navegación por tabs:
  - **Info**: Descripción, dirección, horarios, contacto
  - **Servicios**: Lista completa categorizada con precios y duración
  - **Equipo**: Profesionales con foto, especialidades y calificación
  - **Reseñas**: Calificación global y reseñas verificadas
- Botón flotante "Reservar ahora" visible siempre (FAB en móvil)
- Indicador de horario actual: "Abierto ahora" / "Cerrado" con color semántico
- Galería de fotos con scroll horizontal en móvil

**Interacción móvil**: Scroll vertical con tabs fijos en la parte superior

### Paso 3: Selección de Servicio

**Pantalla**: Selector de servicios

- Lista de servicios agrupados por categoría:
  - Cortes, Barba, Paquetes, Tratamientos, Otros
- Cada servicio muestra:
  - Nombre
  - Duración estimada (icono de reloj)
  - Precio en COP (formato legible: "$25.000")
  - Descripción breve (expandible)
  - Badge de "Popular" o "Nuevo" según datos
- Filtro rápido por categoría (chips horizontales scrollables)
- Selección múltiple permitida (para combos)
- Resumen lateral/inferior con servicios seleccionados y total
- Validación: servicios compatibles con los profesionales disponibles

**Interacción móvil**: Cards apiladas verticalmente, chip scroll horizontal para categorías

### Paso 4: Selección de Profesional

**Pantalla**: Selector de profesional

- Opciones:
  - "Cualquier profesional disponible" (recomendado, faster booking)
  - Lista de profesionales que ofrecen los servicios seleccionados
- Cada profesional muestra:
  - Foto circular
  - Nombre
  - Especialidades relevantes
  - Calificación promedio
  - Próxima disponibilidad ("Tiene cupo hoy a las 3:00 PM")
- Indicador de preferencia si es un profesional recurrente del cliente
- Si no hay profesionales disponibles para los servicios, mostrar alternativas:
  - Otros profesionales con horario similar
  - Sugerir cambiar fecha/hora

**Interacción móvil**: Cards horizontales con scroll, tap para seleccionar

### Paso 5: Selección de Fecha y Hora

**Pantalla**: Calendario de disponibilidad

- Calendario mensual con indicadores:
  - Días con disponibilidad: color primario
  - Días sin disponibilidad: gris deshabilitado
  - Día seleccionado: destacado
- Al seleccionar un día, mostrar slots disponibles:
  - Vista de timeline vertical (móvil) o grid (desktop)
  - Slots agrupados por mañana/tarde/noche
  - Cada slot muestra hora de inicio
  - Indicador de "Último cupo" para slots con poca disponibilidad
  - Color semántico: disponible (verde), ocupado (gris), casi lleno (amarillo)
- Duración total calculada automáticamente según servicios seleccionados
- Advertencia si el horario seleccionado está próximo (ej: "Solo quedan 30 min")

**Interacción móvil**: Scroll vertical por slots, tap para seleccionar

### Paso 6: Confirmación de Reserva

**Pantalla**: Resumen y confirmación

- Resumen completo:
  - Nombre y foto del negocio
  - Servicios seleccionados con precio individual y total
  - Profesional asignado
  - Fecha y hora formateadas (formato largo en español)
  - Duración total estimada
  - Precio total destacado
- Campo opcional: "Notas para el profesional" (textarea)
- Opción de recordatorio: checkbox "Recordarme por WhatsApp"
- Botón "Confirmar reserva" destacado (color primario, full width en móvil)
- Términos y condiciones aceptados implícitamente al confirmar

**Validaciones**:

- Verificar disponibilidad en tiempo real antes de confirmar
- Si el slot fue tomado mientras el usuario seleccionaba, mostrar error amigable
  con alternativas cercanas

### Paso 7: Confirmación Recibida

**Pantalla**: Confirmación de reserva + Notificación

- Pantalla de éxito con:
  - Checkmark animado
  - Número de confirmación
  - Resumen de la reserva
  - Botón "Agregar al calendario" (Google Calendar, Apple Calendar)
  - Botón "Compartir reserva"
- Notificación push/email/WhatsApp enviada simultáneamente
- Acciones rápidas:
  - "Ver mis reservas"
  - "Reservar otro servicio"
  - "Volver al inicio"

### Estados de Error en el Flujo

| Error                     | Mensaje                                                             | Acción               |
| ------------------------- | ------------------------------------------------------------------- | -------------------- |
| Slot ocupado              | "Este horario acaba de ser reservado. Te mostramos otras opciones." | Redirigir a paso 5   |
| Profesional no disponible | "El profesional no está disponible en esta fecha."                  | Sugerir alternativas |
| Negocio cerrado           | "Este negocio no atiende los [día]."                                | Sugerir días hábiles |
| Sesión expirada           | "Tu sesión expiró. Inicia sesión para continuar."                   | Modal de login       |
| Error de red              | "No pudimos completar la reserva. Intenta de nuevo."                | Botón de reintento   |

## Flujo de Onboarding del Negocio

### Journey Completo

```
Registro -> Crear negocio -> Configurar servicios -> Agregar equipo
-> Definir horarios -> Publicar perfil
```

### Paso 1: Registro

**Pantalla**: `/register`

- Formulario simplificado:
  - Nombre completo
  - Email
  - Contraseña (con indicador de fortaleza visual)
  - Teléfono (con código de país, Colombia +57 por defecto)
- Checkbox: "Acepto términos y condiciones" (enlace al documento)
- Botón "Crear cuenta"
- Opción: "Registrarse con Google" (OAuth)
- Verificación por email con enlace de activación
- Redirección automática al wizard de creación de negocio

### Paso 2: Crear Negocio

**Pantalla**: Wizard - Paso 1/5

- Tipo de negocio (selector visual con iconos):
  - Barbería (icono de tijeras)
  - Salón de belleza (icono de espejo)
  - Spa (icono de gota)
  - Otro (icono de configuración)
- Nombre del negocio
- Subdominio deseado (verificación de disponibilidad en tiempo real):
  - Input con sufijo `.beautyspot.co`
  - Feedback instantáneo: disponible (verde) / no disponible (rojo)
- Dirección completa:
  - Autocompletado con Google Places API
  - Coordenadas geográficas capturadas automáticamente
- Teléfono del negocio
- Descripción breve (máximo 300 caracteres)
- Foto de perfil del negocio (upload con preview y recorte)
- Foto de portada (upload con preview)

### Paso 3: Configurar Servicios

**Pantalla**: Wizard - Paso 2/5

- Opción de importar servicios predefinidos por tipo de negocio:
  - Template de barbería: Corte clásico, Corte + Barba, Afeitado, etc.
  - Template de salón: Corte mujer, Tinte, Manicura, Pedicura, etc.
  - Template de spa: Masaje relajante, Facial, Corporal, etc.
- Cada servicio se configura con:
  - Nombre
  - Categoría (dropdown con categorías predefinidas)
  - Precio en COP (input numérico con formateo automático)
  - Duración en minutos (slider o input, intervalos de 15 min)
  - Descripción (opcional)
- Se pueden agregar, editar, eliminar y reordenar servicios
- Vista previa de cómo se verán los servicios en el perfil público
- Botón "Agregar servicio personalizado" para servicios fuera del template

### Paso 4: Agregar Equipo

**Pantalla**: Wizard - Paso 3/5

- Formulario para cada profesional:
  - Nombre completo
  - Email (para invitación a la plataforma)
  - Teléfono
  - Foto de perfil
  - Especialidades (multi-select de servicios configurados en paso anterior)
  - Rol: Profesional, Recepcionista, Administrador
- Lista de profesionales agregados con opción de editar/eliminar
- Invitación por email al profesional con enlace de registro
- Opción "Agregar después" para posponer este paso
- El owner puede auto-asignarse como profesional si también atiende clientes

### Paso 5: Definir Horarios

**Pantalla**: Wizard - Paso 4/5

- Horario general del negocio:
  - Grid semanal con hora de apertura y cierre por día
  - Días de descanso marcados como "Cerrado"
  - Intervalo de citas (15, 30, 45, 60 minutos)
- Horarios individuales de profesionales (opcional):
  - Heredar horario del negocio por defecto
  - Personalizar para cada profesional
  - Excepciones: vacaciones, días libres específicos
- Vista previa del calendario semanal resultante
- Bloqueos iniciales: opción de bloquear fechas específicas

### Paso 6: Publicar Perfil

**Pantalla**: Wizard - Paso 5/5

- Vista previa completa del perfil público:
  - Cómo lo verán los clientes
  - Verificar que toda la información esté completa
- Checklist de elementos configurados:
  - [x] Información básica del negocio
  - [x] Al menos un servicio configurado
  - [ ] Al menos un profesional agregado (recomendado)
  - [x] Horarios definidos
  - [ ] Foto de perfil (recomendado)
- Indicador de "Completitud del perfil" (porcentaje)
- Botón "Publicar mi negocio" (habilitado solo si los campos requeridos están completos)
- Opción "Publicar después" para guardar como borrador
- Al publicar:
  - Perfil visible en el marketplace
  - URL pública activa: `https://{subdominio}.beautyspot.co`
  - Email de bienvenida con guía de inicio rápido

### Post-Onboarding

- Tour guiado interactivo del dashboard (5 pasos clave)
- Tooltip de características no descubiertas
- Sugerencias proactivas: "Agrega más servicios para atraer más clientes"
- Email de seguimiento a los 3 días con tips de configuración

## Flujo de Onboarding del Cliente

### Journey Completo

```
Registro -> Explorar -> Buscar negocio -> Primera reserva
```

### Paso 1: Registro

**Pantalla**: `/register` (variante cliente)

- Formulario mínimo:
  - Nombre
  - Email
  - Contraseña
- Opción: "Registrarse con Google"
- Opción: Registro desde la reserva (si llega desde el marketplace, se le pide
  registro solo al confirmar la primera cita)
- Sin verificación de email obligatoria para clientes (se envía email de bienvenida
  pero no bloquea el acceso)
- Redirección al marketplace con sugerencias personalizadas

### Paso 2: Exploración

**Pantalla**: Home del cliente

- Banner de bienvenida: "Descubre los mejores negocios cerca de ti"
- Solicitar permiso de ubicación para personalizar resultados
- Categorías visuales: Barberías, Salones, Spas (cards con imagen e icono)
- "Populares cerca de ti" (carousel horizontal)
- "Nuevos en la plataforma" (carousel horizontal)
- "Con cupo disponible hoy" (carousel horizontal)
- Sección de promociones activas (si las hay)

### Paso 3: Primera Reserva

- Flujo guiado con tooltips:
  - "Selecciona el servicio que deseas"
  - "Elige a tu profesional o déjalo aleatorio"
  - "Selecciona la fecha y hora que prefieras"
  - "Confirma tu reserva y listo!"
- Post-primera reserva:
  - Pantalla de felicitaciones: "Tu primera reserva esta lista!"
  - Sugerencia de habilitar notificaciones WhatsApp
  - Sugerencia de agregar negocios favoritos

## Estrategia Mobile UX

### Principios Mobile-First

- **Touch-first**: Todos los targets táctiles mínimo 44x44px (Apple HIG)
- **Gestos nativos**: Swipe para navegar entre tabs, pull-to-refresh para actualizar
- **Thumb zone**: Acciones principales en la zona alcanzable con el pulgar
- **Viewport responsivo**: Breakpoints en 640px (sm), 768px (md), 1024px (lg), 1280px (xl)

### Breakpoints y Adaptaciones

| Breakpoint  | Dispositivo                   | Layout                                      |
| ----------- | ----------------------------- | ------------------------------------------- |
| < 640px     | Móvil                         | Stack vertical, navegación bottom tabs, FAB |
| 640-768px   | Móvil grande / Tablet pequeña | Stack vertical, sidebar colapsable          |
| 768-1024px  | Tablet                        | Grid 2 columnas, sidebar fija               |
| 1024-1280px | Laptop                        | Grid 3 columnas, sidebar expandida          |
| > 1280px    | Desktop                       | Grid completo, multi-panel                  |

### Navegación Móvil

**Bottom Navigation Bar** (cliente):

- Home (icono casa)
- Buscar (icono lupa)
- Mis Reservas (icono calendario)
- Favoritos (icono corazón)
- Perfil (icono usuario)

**Bottom Navigation Bar** (profesional):

- Agenda (icono calendario)
- Citas (icono lista)
- Disponibilidad (icono reloj)
- Perfil (icono usuario)

**Sidebar + Hamburger** (admin/owner):

- Dashboard
- Citas
- Clientes
- Servicios
- Equipo
- Finanzas
- Marketplace
- Configuración

### Optimizaciones para Móvil

- **Lazy loading** de imágenes con blur-up placeholder
- **Skeleton screens** durante carga (no spinners)
- **Offline banner** cuando no hay conexión
- **Swipe actions** en listas (swipe izquierda para cancelar cita, derecha para confirmar)
- **Haptic feedback** en acciones importantes (confirmación, error)
- **Share API** nativa para compartir perfil de negocio o reserva
- **Add to Home Screen** (PWA manifest completo)

### Progressive Web App (PWA)

- Service Worker para caché de assets estáticos y datos frecuentes
- Manifest con iconos adaptativos para Android e iOS
- Notificaciones push (Firebase Cloud Messaging)
- Soporte offline para: ver reservas pendientes, perfil del usuario, favoritos
- Instalable desde el navegador sin app store

```
--space-6:  1.5rem    (24px)  - márgenes entre secciones
--space-8:  2rem      (32px)  - padding de páginas
--space-10: 2.5rem    (40px)  - separación de bloques grandes
--space-12: 3rem      (48px)  - hero spacing
--space-16: 4rem      (64px)  - separación entre páginas
```

### Componentes Base

#### Botones

| Variante  | Uso                                          | Estilo                         |
| --------- | -------------------------------------------- | ------------------------------ |
| Primary   | Acción principal (Confirmar, Guardar)        | Filled violeta, texto blanco   |
| Secondary | Acción alternativa (Cancelar, Volver)        | Outline violeta, texto violeta |
| Ghost     | Acción terciaria (Filtrar, Más opciones)     | Sin borde, texto gris          |
| Danger    | Acción destructiva (Eliminar, Cancelar cita) | Filled rojo, texto blanco      |
| Success   | Confirmación positiva (Completar)            | Filled verde, texto blanco     |

Tamaños: `sm` (32px), `md` (40px), `lg` (48px)
Ancho completo en móvil (`w-full`) para CTAs principales

#### Cards

- Bordes redondeados: `rounded-xl` (12px)
- Sombra sutil: `shadow-sm` por defecto, `shadow-md` en hover
- Padding: `p-4` en móvil, `p-6` en desktop
- Hover: elevación de sombra + borde sutil
- Transición: `transition-all duration-200`

#### Inputs

- Altura: 44px (touch-friendly)
- Bordes: 1px solid gray-300, focus: 2px solid primary-500
- Labels encima del input (no como placeholder)
- Mensaje de error debajo del input con color error
- Icono decorativo opcional a la izquierda

#### Modales

- Overlay oscuro con blur (backdrop-blur-sm)
- Centrado vertical y horizontal
- Ancho máximo: 480px en móvil (full width), 600px en desktop
- Animación: fade-in + slide-up
- Cerrar con botón X, click fuera, o tecla Escape

#### Toasts / Notificaciones

- Posición: top-right en desktop, top-center en móvil
- Duración: 5 segundos (success), 7 segundos (error)
- Tipos: success (verde), error (rojo), warning (amarillo), info (azul)
- Acción opcional: botón "Deshacer" o "Ver detalle"

## Consideraciones de Accesibilidad

### Estándar Objetivo

WCAG 2.1 nivel AA como mínimo. Las áreas críticas (reserva, onboarding)
buscarán nivel AAA.

### Pautas Implementadas

#### Perceptible

- Contraste de color mínimo 4.5:1 para texto normal, 3:1 para texto grande
- No depender solo del color para transmitir información (usar iconos + texto)
- Texto alternativo en todas las imágenes significativas
- Formatos de fecha y hora legibles por pantalla (no solo relativos)
- Labels visibles para todos los campos de formulario

#### Operable

- Navegación completa por teclado (Tab, Enter, Escape, flechas)
- Skip navigation link al inicio de cada página
- Focus visible con outline claro (no `outline: none`)
- Áreas de click mínimo 44x44px
- Sin timeouts en formularios (permitir completar a ritmo propio)
- Sin contenido que parpadee más de 3 veces por segundo

#### Comprensible

- Idioma del documento declarado: `<html lang="es">`
- Labels claros y descriptivos para todos los inputs
- Mensajes de error específicos y con sugerencia de corrección
- Navegación predecible y consistente entre páginas
- Confirmación antes de acciones destructivas (cancelar cita, eliminar servicio)

#### Robusto

- HTML semántico: `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`
- ARIA labels en elementos interactivos personalizados
- ARIA live regions para contenido dinámico (nuevas notificaciones, errores)
- Roles ARIA donde el HTML semántico no alcanza (tabs, modales, dropdowns)

### Testing de Accesibilidad

- Lighthouse Accessibility score >90 en todas las páginas
- Pruebas manuales con lector de pantalla (NVDA/VoiceOver) en flujos críticos
- Pruebas de navegación por teclado en cada release
- Axe Core integrado en tests E2E

## Patrones de Interacción

### Estados de Carga

- **Skeleton screens**: Usar placeholders con animación shimmer para contenido que carga
- **Botones con loading**: Deshabilitar y mostrar spinner durante la acción
- **Carga progresiva**: Mostrar contenido parcial mientras carga el resto
- **Pull-to-refresh**: En listas móviles, permitir recargar con gesto

### Estados Vacíos

Cada pantalla con datos dinámicos debe tener un estado vacío ilustrado:

- Sin citas: "No tienes citas programadas. Reserva ahora!"
- Sin servicios: "Agrega tu primer servicio para empezar a recibir reservas"
- Sin reseñas: "Aun no tienes reseñas. Tus clientes podran dejarte una tras su cita"
- Sin clientes: "Tus clientes apareceran aqui tras su primera cita"

Cada estado vacío incluye un CTA que guía al usuario a la acción siguiente.

### Confirmaciones

| Acción               | Tipo de Confirmación                        |
| -------------------- | ------------------------------------------- |
| Cancelar cita        | Modal con motivo de cancelación (requerido) |
| Eliminar servicio    | Modal con advertencia sobre citas futuras   |
| Eliminar profesional | Modal con opción de reasignar citas         |
| Cerrar caja          | Modal con resumen de movimientos            |
| Modificar horario    | Inline warning sobre citas afectadas        |

### Feedback de Acciones

| Acción             | Feedback                                                |
| ------------------ | ------------------------------------------------------- |
| Reserva creada     | Toast success + animación de checkmark + redirección    |
| Cita confirmada    | Toast success + actualización de estado en calendario   |
| Pago registrado    | Toast success + actualización de saldo                  |
| Error de red       | Toast error persistente con botón "Reintentar"          |
| Validación fallida | Mensaje inline debajo del campo con corrección sugerida |
| Sesión expirada    | Modal de re-autenticación sin perder contexto           |

## Referencias de Diseño

### Inspiración Visual

- **Calendario**: Calendly / Acuity Scheduling (limpieza y claridad)
- **Dashboard**: Linear (minimalismo funcional) / Notion (flexibilidad)
- **Marketplace**: Treatwell / Booksy (cards de negocio con ratings)
- **Onboarding**: Stripe (wizard progressivo con pasos claros)
- **Sistema de colores**: Tailwind UI (coherencia y accesibilidad)

### Iconografía

- Librería: Heroicons (outline style) por defecto
- Tamaño estándar: 20px en UI, 24px en navegación
- Iconos coherentes: mismo estilo (outline) en toda la aplicación
- No mezclar librerías de iconos

### Ilustraciones

- Estilo: Flat/semi-flat con colores de la paleta primaria
- Uso: estados vacíos, onboarding, errores 404/500
- No usar ilustraciones en áreas de alta densidad de información
